# Basegame Simulator README

이 문서는 **새 채팅 부트스트랩용 단일 진실 소스**다.  
목표는 이 README 하나만 보면, 프로젝트 개요 / 핵심 룰 / 승리 조건 / AI 구조 / 최근 실험 결과 / 남은 과제까지 복구할 수 있게 하는 것이다.

## 2. 이 프로젝트의 목적

이 시뮬레이터는 다음을 목적으로 한다.

1. 보드게임 룰을 코드로 정확히 재현한다.
2. 다수 게임(10판, 100판, 그 이상)을 자동 실행해 메타를 분석한다.
3. 인물/잔꾀/날씨/운수/주사위 카드/보드 구조가 게임 길이와 승률에 주는 영향을 확인한다.
4. AI 정책을 바꿔 가며 “밸런스가 맞는가, 너무 빨리 끝나는가, 특정 전략이 과도하게 강한가”를 검증한다.
5. 보드/룰/정책을 외부 파일로 분리해서 실험 가능한 구조를 유지한다.

## 3. 문서 체계와 작업 규칙

이 프로젝트는 **문서 우선**이다.

반드시 기억할 것:
- 작업 시작 전 `MODULE_READ_ORDER.md`를 먼저 본다.
- 기본 원칙은 **`*.md` 먼저, 그 다음 대응 `*.py`** 다.
  - 예: `engine.md -> engine.py`, `ai_policy.md -> ai_policy.py`
- 소스를 수정하면 대응 문서도 같이 수정해야 한다.
- `doc_integrity.py`가 **문서 수정 시각 >= 소스 수정 시각**을 검사한다.
- 관련 테스트(`test_doc_integrity.py`, `test_config_settings.py` 등)에도 무결성 검사가 연결돼 있다.

즉, 이 프로젝트는 “코드만 맞으면 됨”이 아니다.  
**코드, 문서, 로그 스키마, 메타데이터가 같이 맞아야 한다.**

## 4. 현재 파일 구조에서 중요한 모듈

### 핵심 런타임
- `engine.py`: 턴 진행, 이동, 도착 처리, 종료 판정, 결과 생성
- `event_system.py`: 의미 단위 이벤트 dispatch
- `effect_handlers.py`: 실제 효과 처리 로직
- `state.py`: 게임 상태, 플레이어 상태, 타일 상태
- `config.py`: 전역 설정 mirror / 기본 구성
- `game_rules.py`: 룰 주입 모델
- `game_rules_loader.py`: ruleset JSON 로더
- `rule_script_engine.py`: JSON rule script override 처리

### AI / 정책
- `ai_policy.py`: 인물 선택 / 이동 / 구매 / 잔꾀 / 랩 보상 / 지목 등 휴리스틱
- `policy_hooks.py`, `policy_mark_utils.py`, `policy_groups.py`: 정책 보조 로직

### 보드
- `board_layout.json`, `board_layout.csv`, `board_layout_meta.json`: 외부화된 보드 정의
- `board_layout_creator.py`: 구조 로딩/생성기

### 카드/데이터
- `characters.py`
- `fortune_cards.py` + `fortune.csv`
- `trick_cards.py` + `trick.csv`
- `weather_cards.py` + `weather.csv`

### 실행 / 분석
- `simulate_with_logs.py`: 배치 시뮬레이션 및 JSONL/summary 출력
- `run_chunked_batch.py`: 청크 단위 장기 실행
- `analyze_strategy_logs.py`: 로그 후처리 분석
- `debug_cli.py`: 디버그 실행
- `print_settings.py`: 현재 설정 출력

## 5. 베이스게임 핵심 룰 요약

## 5-1. 현재 기본값과 실험 프리셋 구분

- `ruleset.json` / `GameConfig()` 기본값은 **실제 코드가 바로 사용하는 기준값**이다.
- README/대화/분석 로그에 남아 있는 30/40/50냥, 시작 코인 2~3개 같은 수치는 **실험 프리셋 또는 과거 브랜치 결과**일 수 있다.
- 따라서 기본값을 확인할 때는 항상 `ruleset.json`, `config.py`, `test_config_settings.py`를 함께 본다.


### 플레이어 수
- 기본: **4인**

### 시작 자원 (현재 ruleset / runtime 기본값 기준)
- 시작 돈: **20냥**
- 시작 손 코인(승점 토큰): **0개**
- 시작 미리내 조각: **2개**
- 시작 보유 타일: **없음**
- 추가 시작 재화 선택 메타데이터: **20PTS** 안에서 현금(2PT), 조각(3PT), 승점 토큰(3PT) 선택

> 과거 실험 로그에는 30냥 / 40냥 / 50냥, 시작 코인 2~3개 같은 프리셋이 등장한다.
> 그러나 **현재 업로드본의 기본 런타임 값은 `ruleset.json` / `GameConfig` 기준 20냥 / 0코인 / 2조각**이다.

### 랩 보상
기본적으로 랩 시 아래 중 선택한다.
- 현금: **5냥**
- 코인: **3개**
- 조각: **3개**

### 종료 조건 (현재 ruleset 기준)
다음 중 하나 충족 시 게임 종료:
- `F_THRESHOLD`: **F 15 이상**
- `THREE_MONOPOLIES`: **독점 3개**
- `NINE_TILES`: **타일 9개 이상**
- `ALIVE_THRESHOLD`: **생존 플레이어 수가 2명 이하**

즉, 승리는 단순 “돈 제일 많음”이 아니라 **여러 종료축 중 어떤 축이 먼저 열리느냐**에 따라 갈린다.

## 6. 보드 구조

### 전체 구조
- 총 칸 수: **40칸 외곽 순환형**
- 특수칸 분포:
  - `F1`: 2칸
  - `F2`: 2칸
  - `S`: 8칸
  - `T2`: 24칸
  - `T3`: 4칸

### 코너 구조
- `0 = F1`
- `10 = F2`
- `20 = F1`
- `30 = F2`

### 일반 패턴
- F1/F2 사이마다 9칸
- 12개 블록(block)으로 구성
- 3칸짜리 구역은 `B2, B5, B8, B11`이며 패턴은 `T2-T3-T2`
- 나머지는 2칸짜리 구역 `T2-T2`

### 색상 순환
- `검은색 -> 빨간색 -> 노란색 -> 파란색 -> 하얀색 -> 초록색 -> 반복`

### 경제 프로필 (v0.7.61 구조)
보드는 이제 숫자를 직접 박지 않고 `economy_profile`을 참조한다.

- `HIGH`: 구매 5 / 통행료 5
- `MID`: 구매 4 / 통행료 4
- `LOW`: 구매 3 / 통행료 3

악성 토지 비용:
- 기존 땅 가격 x 3
  - HIGH -> 15
  - MID -> 12
  - LOW -> 9

### 특수칸 규칙
#### F1
- F +1
- 조각 +1

#### F2
- F +2
- 조각 +2

#### S(운수)
- 50%: +1냥
- 25%: +2냥
- 25%: -1냥

## 7. 코인(승점 토큰) 규칙

현재 룰셋 기준:
- 시작 손 코인: 0개
- 랩 보상 코인: 3개
- 자기 땅 방문 시 손 코인 +1
- 타일당 최대 배치: 3개
- 1회 방문당 최대 배치: 3개
- 첫 구매 직후에도 배치 가능
- 첫 구매 시 최대 배치 수: 1개
- 인수 시 타일 위 코인도 함께 이전 가능

핵심 의도:
- 코인을 손에 들고만 끝내지 않고, 내 땅 재방문을 통해 보드에 배치하게 만들고 싶다.
- 다만 과거 실험에서는 **게임이 너무 빨리 끝나 손 코인이 많이 남는 문제**가 있었다.

## 8. 주사위 카드 규칙

현재 룰셋 기준:
- 숫자 카드: **[1] [2] [3] [4] [5] [6]** 각 1장
- 일회용
- 사용한 숫자는 `used_dice_cards`에 들어가며 소모됨
- 턴당 최대 사용 수: **2장**
- 사용 방식: **카드 1장 + 실제 주사위 1개** 허용
- `카드 2장 합산 고정` 같은 구조가 아니라, 현재 설계는 카드와 실제 주사위 혼합 사용이 핵심이다.

AI 관점에서 주사위 카드는 매우 중요하다.
- 구매
- F1/F2/S 진입
- 렌트 회피
- 자기 땅 재방문
- 강제이동/견제 대응

등에 적극 활용된다.

## 10. 잔꾀 / 운수 / 날씨의 핵심 성격

### 잔꾀(trick)
- 기본적으로 자기 턴에는 **잔꾀 먼저**, 그 다음 일반 행동 흐름
- 일반 잔꾀 사용은 1장 제한 흐름을 기본으로 유지해 왔다
- 성장형 / 이동형 / 렌트 제어형 / 방어형 / 부담(짐)형 카드가 섞여 있다
- 현재 메타에서 특히 중요했던 카드 예시:
  - `건강 검진`: 전체 통행료 반감
  - `우대권`: 통행료 1회 무시
  - `무료 증정`: 무료 구매
  - `마당발`: 추가 인접 구매
  - `강제 매각`: 상대 토지 강제 매각
  - `저속`: 돈 받고 주사위 1개 줄이기
  - `과속`: 돈 내고 주사위 1개 늘리기
  - `무거운 짐`, `가벼운 짐`: burden 계열 리스크 카드

### 운수(fortune)
- 이동/인수/매각/과징금/보상/짐 처리/즉시 이동 등 변동성이 큼
- 돈을 직접 깎거나, 토지를 뺏거나, 짐을 처리/누적시키는 카드가 있어 생존 점수에 중요하다.

### 날씨(weather)
- 전역 효과
- 렌트 2배 색상 강화
- 전체 과금
- 징표 관련 효과
- 긴급 피난 / 종료 당김 / 잔꾀 보충 등
- 특히 `긴급 피난`은 burden cleanup 비용을 강제로 발생시켜 파산 원인으로 자주 등장했다.

## 12. 인물별 전략적 의미(실전 해석)

이 섹션은 **정확한 카드 문구 전체**가 아니라, AI/메타 관점에서의 핵심 의미를 정리한 것이다.

### 어사
- 무뢰 속성 비활성화 성격
- 상대 무뢰 메타를 꺾는 안정형 통제 카드

### 탐관오리
- 관원/상민 상대로 돈 압박을 거는 **돈 뺏는 인물**
- 공개정보 기반 생존판단에서 **활성면이면 money distress를 올리는 위험 신호**로 취급

### 자객
- 지목 기반 차단 카드
- 저현금이어도 기능 가능한 견제형

### 산적
- 지목 성공 시 현금 압박
- 활성 상태면 상대 생존 계산에서 주의해야 하는 돈 압박 카드

### 추노꾼
- 지목 대상 강제이동
- 강제이동 뒤 렌트/특수칸 효과가 정상 적용됨
- 랩 보상 없음
- 활성 상태면 전방 위험과 함께 **2턴 내 사망 리스크**를 키우는 요소

### 탈출 노비
- 시작/F/운수 근처로 보정 도착 가능
- 생존형 / 탈출형 / 저현금 대응형 핵심 인물

### 파발꾼
- 기동력, 주사위 추가, 추가 이동
- 생존/돈벌이/템포 확보에 유용

### 아전
- 특정 상황에서 통행료를 피하거나 같은 칸 상호작용으로 돈을 얻는 쪽의 **저현금 기능형**
- 활성 상태면 상대에게도 돈 압박 신호가 될 수 있음

### 교리 연구관 / 교리 감독관
- 징표 방향 제어
- 최근 AI 패치에서는 **감독관 계열을 통해 돈 압박 인물을 비활성화하는 선택**이 중요해졌다.
- 즉 단순 점수 카드가 아니라 **컨트롤/방어적 메타 대응 카드**다.

### 박수
- 짐을 넘기고 잔꾀를 가져오는 burden 재배치 카드
- 공개 burden pressure가 높을수록 가치 상승

### 만신
- 상대 burden 정리 + 비용 지불 강요
- 상대 짐이 많은 경우 돈 압박 카드가 됨
- burden 보유 상태에서는 생존 계산에 반드시 반영해야 하는 공개위험 요소

### 객주
- 랩/F 경유 강화 + 자기 땅 도착 시 코인 수급
- 돈/조각/코인 모두 만질 수 있는 성장-생존 혼합형
- 저현금에서도 비교적 기능하는 편

### 중매꾼
- 인접 같은 색 구역 추가 매입
- **실제 운용 자금이 있어야만 의미가 큰 성장형**
- 저현금 상태에서 고르면 능력을 못 살리고 맞아 죽기 쉬움
- 최근 패치에서 강하게 감점 대상

### 건설업자
- 무료 구매 기반 확장형
- 견제가 집중되는 환경에서는 위험
- 저현금/위기 상태에서는 피해야 할 성장형

### 사기꾼
- 통행료의 2배를 내고 토지를 인수
- 인수 시 해당 타일에 적립된 코인도 함께 가져감
- **탈취 성공 기대값은 높지만, 비용 실패 시 SWINDLE_FAIL로 폭사 위험**이 큼
- 최근 분석에서 첫 사망자의 절반 가까이가 사기꾼이었다.

## 13. 반드시 기억할 최신 룰 보정

과거 대화에서 계속 유지되어야 하는 핵심 보정들:

### 중매꾼
- 구역 내 **인접 토지 1개만** 추가 구매 가능
- **도착 칸이 T2/T3일 때만** 발동

### 추노꾼
- 강제이동 자체는 정상 룰
- 강제이동 후 렌트/특수칸 효과 적용
- 랩 보상 없음

- **독점 구역은 어떤 수단으로도 인수 불가**
- 인물 / 운수 / 잔꾀 모두 포함

- 카드 1장 제외
- 4장 풀 드래프트
- 마지막 플레이어는 남은 1장 + 제외 3장 섞은 4장 중 1장 추가

- 일반 잔꾀는 1장 사용 흐름을 기본으로 유지

### 객주

### 박수 / 만신
- 짐 카드 기준 최신 능력으로 교체
- burden pressure 반영이 중요

## 14. 승리 구조를 이해하는 법

이 게임은 단순 점수합만 보면 안 된다.  
실제로는 아래 네 축의 레이스다.

1. **F 레이스**
2. **타일/독점 레이스**
3. **생존 레이스**
4. **코인/조각/현금 엔진 유지 레이스**

특정 패치나 설정은 “게임이 길어졌다”가 아니라,
- 파산형 종료가 줄고 F 종료가 늘 수 있고
- 타일 폭주가 줄고 생존형 종료가 늘 수 있고
- 코인 엔진이 살아도 재방문 전에 끝날 수 있다.

따라서 실험 해석은 항상 다음을 같이 봐야 한다.
- 평균 총 턴
- 종료 사유 분포
- 평균 F
- 평균 배치 코인 / 손에 남은 코인
- 파산 게임 비율 / 플레이어 단위 파산 비율
- 정책별 승률
- 정책별 랩 보상 선택 비율

## 16. 과거 AI의 핵심 문제

예전에는 이런 문제가 컸다.

### 1) 메타가 너무 빨리 끝남
- 4인 기준 평균 20턴대
- 플레이어당 체감 5턴 수준
- 코인을 얻어도 내 땅 재방문 전에 게임 종료

### 2) 성장형 인물 과대평가
- 저현금인데도 `중매꾼`, `건설업자`, `사기꾼`을 고름
- 능력을 못 살리고 견제/렌트/오프턴 과금에 죽음

### 3) 사기꾼 선택 문제
- 사기꾼은 `유효 통행료 x 2` 지불이 필요함
- 돈이 부족한데도 가치가 높게 계산되어 고르는 경우가 있었음
- `SWINDLE_FAIL`이 파산의 큰 축으로 등장

### 4) 생존 판단 horizon이 짧음
- 단순히 “다음 턴에 죽나” 정도만 봐서는 부족
- 실제로는 **다다음 턴(2턴 horizon)**, 공개된 위협, 액티브 인물, 공개 잔꾀, 전방 타일까지 봐야 했다.

## 17. 현재 AI 설계의 핵심: 범용 생존 점수

최근 패치의 가장 중요한 설계 원칙:

> **생존은 최우선이다.**

그리고 이 생존 판단은 개별 선택마다 따로 노는 게 아니라,
- 인물 선택
- 잔꾀 선택
- 주사위 카드 선택
- 랩 보상 선택
- 구매 여부
- 감독관 flip

모두가 참조하는 **공통 컨텍스트**여야 한다.

### generic survival score가 보는 것
- 유동성 reserve
- 렌트 압박
- 즉사 확률
- burden cleanup 압박
- 시작점/F/S 접근성
- 공개된 전방 위험 타일
- 활성면 돈 압박 인물
- 공개 잔꾀
- 전체 플레이어 현금 분포

이 컨텍스트에서 아래 파생 신호를 만든다.
- `money_distress`
- `two_turn_lethal_prob`
- `controller_need`
- `reserve_gap`
- `rent_pressure`
- `burden_pressure`

## 18. money distress(돈이 부족하다) 신호의 의미

이 신호는 단순히 “현재 현금 수치가 낮다”가 아니다.  
**현재 공개정보 전체를 보고, 돈이 부족한 상태인지**를 판정한다.

반영해야 하는 대표 조건:
- 짐이 많다
- 앞쪽 타일들이 남의 타일로 가득 차 있다
- 활성화된 인물 중 돈을 빼앗는 면이 켜져 있다
  - 탐관오리
  - 산적
  - 아전
  - 추노꾼
  - burden 상태에서의 만신
- 공개 잔꾀나 날씨가 돈 압박을 유발한다
- 내 위치, 남의 위치, 타일 색, 날씨가 전방 렌트/이동 위험을 높인다

핵심은:
- “다음 턴에 죽을 수 있나?”만이 아니라
- **“다다음 턴까지 포함하면 위험한가?”** 를 보는 것이다.

---

## 19. 생존 위기일 때 AI가 해야 하는 행동

돈이 부족할 때 해야 할 것은 단순한 “절약”이 아니다.  
**현재 상황을 타개하는 선택**이 필요하다.

### 우선 행동 원칙
1. 돈을 벌 수 있는 인물을 고른다.
2. 구매를 잠시 미룬다.
3. 현재 상황을 탈출할 잔꾀를 찾는다.
4. 필요하면 감독관으로 돈 압박 인물을 끈다.
5. F1/F2/S/시작점 같은 회복 동선을 선호한다.

### AI에 반영된 대응 예시
- 저현금이면 `객주`, `아전`, `만신`, `탈출 노비`, `파발꾼` 가치 상승
- 저현금이면 `중매꾼`, `건설업자`, `사기꾼` 강한 감점
- 감독관 계열은 `controller_need`가 높을수록 가산
- 랩 보상은 위기 시 현금 선호 증가
- 주사위 카드는 F/S/자기 생존동선 진입용 선호 증가
- 타일 구매는 생존선 유지가 안 되면 스킵

## 20. purchase gate와 end-of-turn safety의 의미

현재 문제를 이해하려면 **“내 턴 행동”보다 “내 턴 종료 상태”**를 봐야 한다.

최근 분석에서 드러난 사실:
- 첫 사망자의 다수는 **오프턴 사망**이었다.
- 즉, 자기 턴에 바로 죽는 게 아니라
  - 턴 끝날 때 현금이 너무 얇고
  - 그 다음 외부 이벤트 한 번에 죽는다.

따라서 purchase gate의 진짜 의미는:
- 이번 턴 구매할 수 있는가?
가 아니라
- **구매하고 나서도 오프턴 과금 / 렌트 / 강제이동 / burden cleanup을 버틸 수 있는가?**
를 보는 것이다.

향후 더 강화해야 할 방향:
- end-of-turn reserve floor
- burden cleanup 가상 부채 반영
- 사기꾼 탈취 후 생존선 유지 여부 검사

## 21. 사기꾼(SWINDLE_FAIL) 관련 중요 메모

### 의미
`SWINDLE_FAIL`은 “사기꾼이 탈취에 실패한 것”이다.

정확한 흐름:
1. 사기꾼이 남의 땅 도착
2. 유효 통행료 x 2 비용 계산
3. 비용 지불 성공 시 `SWINDLE_TAKEOVER`
4. 비용을 못 내면 `SWINDLE_FAIL`
5. 현금 부족이면 그대로 파산 가능

### 왜 위험한가
- 기대가치는 높다: 땅 + 타일 위 코인 + 독점 저지/완성
- 하지만 비용 실패 시 즉시 위험
- 최근 분석에서 **첫 사망자의 절반 가까이가 사기꾼**이었다.

### 설계 교훈
사기꾼은 “좋은 카드”가 아니라,
- **돈이 충분하고**
- **탈취 후에도 생존 가능한 경우에만 좋은 카드**다.

## 27. ALIVE_THRESHOLD 53판 첫 사망자 분석 결과

이 분석은 `survival_gate` 100판 중 `ALIVE_THRESHOLD`로 끝난 53판에서,
**각 게임의 첫 사망자**를 full log 기준으로 추적한 결과다.

### 핵심 수치
- 첫 사망 평균 시점: **28.87턴**
- 중앙값: **28턴**
- 첫 사망자는 평균 자기 턴 **6.58번** 가진 뒤 탈락
- 오프턴 사망: **46 / 53 = 86.8%**
- 자기 턴 사망: **7 / 53 = 13.2%**

### 즉시 원인 그룹
- `RENT / 강제이동 후 렌트 / 분쟁통행`: **20건**
- `긴급 피난 burden cleanup`: **17건**
- `SWINDLE_FAIL`: **10건**
- 기타 자기부담 비용: **6건**

### 직전 패턴
- 직전 3턴 안에 타일 구매: **39 / 53 = 73.6%**
- 직전 3턴 안에 주사위 카드 사용: **33 / 53 = 62.3%**
- 마지막 자기 턴 종료 현금 10 이하: **23 / 53**
- 마지막 자기 턴 종료 현금 5 이하: **15 / 53**

### 해석
- 첫 사망은 대체로 **내 턴 즉사**가 아니라 **오프턴 취약 상태에서 외부 이벤트 한 번에 죽는 구조**다.
- 즉 생존 문제의 본질은 “이 턴에 뭘 하느냐”보다,
  **“이 턴을 어떤 자원 상태로 끝내느냐”** 에 있다.

### 정책 분포
- `heuristic_v2_control`: 15
- `heuristic_v2_token_opt`: 14
- `heuristic_v2_balanced`: 14
- `heuristic_v1`: 10

### 캐릭터 분포
- `사기꾼`: 25
- `파발꾼`: 9
- `객주`: 4
- `추노꾼`: 4
- `박수`: 4
- `자객`: 3

가장 강한 메시지:
- **사기꾼은 여전히 첫 탈락 리스크가 매우 높은 카드다.**

## 28. 현재 최신 패치(survival_publicinfo_patch)의 설계 목표

이 패치는 아직 100판 재검증 전이지만, 의도는 명확하다.

### 추가된 핵심
- 생존 판단을 **공개정보 기반 2턴 horizon**으로 강화
- money distress를 단순 현금 부족이 아니라 상황 인식 신호로 재정의
- controller_need를 통해 감독관 flip의 전략적 가치 반영
- 돈이 부족하면
  - 돈벌이/탈출형 인물 선호
  - 구매 보류
  - 상황 타개용 잔꾀/동선 선택
  - 감독관으로 돈 압박 인물 비활성화 선호

### 기대 효과
- “저현금인데 성장형 인물 선택” 감소
- “턴 종료 시 현금이 너무 얇은 상태” 감소
- “사기꾼/burden/오프턴 과금으로 인한 첫 사망” 감소

### 아직 남은 검증 과제
- 실제로 100판 기준 파산율이 더 내려가는가?
- 평균 턴이 늘어나는가?
- F 레이스가 과도하게 강해지지는 않는가?
- token_opt/control의 생존성이 개선되는가?
