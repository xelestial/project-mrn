대상 선택 prompt 또는 결과 event를 표시합니다. 이때 마크 효과는 "대상 턴 시작에 해결된다"는 규칙을 기억해야 합니다.

## 상세 인수인계

### 이 문서에서 이해해야 할 것

이 문서는 `턴 시작 마크/지목` 기능을 따라갑니다. 대상 캐릭터를 찍는 효과는 대상 플레이어의 턴 시작에 해결되므로 즉시 효과처럼 다루면 안 됩니다. 기능을 고칠 때는 한 계층만 보지 말고 엔진, 백엔드, 프론트 표시가 같은 흐름을 가리키는지 함께 확인해야 합니다.

### 관련 코드와 문서

- `GPT/runtime_modules/turn_modules.py`
- `GPT/runtime_modules/sequence_modules.py`
- `GPT/runtime_modules/modifiers.py`
- `GPT/policy/character_traits.py`
- `apps/server/src/services/runtime_service.py`

### 실제 처리 흐름과 기대 값

- 이 문서의 흐름: 턴 시작 마크/지목.
- 사용자 행동/트리거: 현재 턴 시작 전 특정 캐릭터/효과가 대상 선택을 요구할 때 발생합니다.
- 프론트 경로: 프론트는 `PromptOverlay`와 `selectMarkTargetCharacterSlots`로 선택 가능한 캐릭터 슬롯을 보여주고, 선택 결과를 WebSocket decision으로 보냅니다.
- 백엔드/엔진 경로: 백엔드는 decision을 접수하고, 런타임은 `turn_modules.py`, `sequence_modules.py`, `modifiers.py`의 턴 시작/마크 모듈에서 성공/실패/보류를 결정합니다.
- 기대 값/화면 결과: 기대 값은 지목 대상의 `character_id` 또는 선택 없음, `mark_queued`/`mark_resolved`/`mark_blocked` 이벤트, 다음 core action 단계입니다. 화면에서는 현재 차례와 대상 표시가 이벤트 로그와 어긋나면 안 됩니다.

### 수정할 때 주의할 점

- 상위 문서와 하위 문서를 함께 읽고, 관련 테스트를 먼저 찾으세요.

### 확인 방법

- `python3 tools/plan_policy_gate.py`
- 관련 서버/프론트/엔진 테스트를 함께 실행하세요.

### 초보자가 자주 헷갈리는 질문

- “이 값은 프론트가 계산하나요?”: 기본적으로 아니요. 규칙/권한/비공개성은 엔진과 백엔드가 결정해야 합니다.
- “화면이 맞으면 구현도 맞나요?”: 아니요. stream event, view_state, 테스트가 같은 흐름을 말해야 합니다.
- “어디부터 로그를 보나요?”: stream `seq`를 먼저 보고, 프롬프트 문제면 `request_id`, 런타임 문제면 `module_id`를 봅니다.
