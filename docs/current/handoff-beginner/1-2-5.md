# 1-2-5. 한 플레이어 턴의 기본 순서

문서 위치: [1. 게임 룰](1.md) > [1-2. 라운드의 기본 흐름](1-2.md) > [1-2-5. 한 플레이어 턴의 기본 순서](1-2-5.md)

한 턴은 대략 다음 순서로 진행됩니다.

1. 턴 시작 전 지목/마크 계열 효과 처리
2. 턴 시작 알림
3. 캐릭터 능력 처리
4. 잔꾀 카드 사용 여부 선택
5. 주사위 또는 주사위 카드 결정
6. 말 이동
7. 도착 칸 처리
8. 턴 종료 스냅샷

## 상세 인수인계

### 이 문서에서 이해해야 할 것

이 문서는 게임 규칙 중 `한 플레이어 턴의 기본 순서`을 설명합니다. 규칙 문서는 UI 문구가 아니라 엔진이 실제로 따라야 하는 계약으로 읽어야 합니다. 화면에서 그럴듯하게 보여도 엔진 이벤트와 테스트가 다르면 규칙 구현이 맞다고 볼 수 없습니다.

### 관련 코드와 문서

- `docs/current/Game-Rules.md`
- `GPT/runtime_modules/round_modules.py`
- `GPT/runtime_modules/turn_modules.py`
- `GPT/engine.py` (모듈에서 아직 호출하는 레거시 규칙 helper 확인용)
- `GPT/test_rule_fixes.py`

### 실제 처리 흐름과 기대 값

- 이 문서의 규칙 흐름: 한 플레이어 턴.
- 게임 안에서 시작되는 지점: 현재 actor가 정해지면 턴 시작 효과, 잔꾀/주사위, 이동, 도착 처리, 턴 종료가 순서대로 진행됩니다.
- 처리 방식: 선택이 필요한 단계만 prompt/decision으로 멈추고, 자동 단계는 엔진이 event와 view_state를 연속 발행합니다.
- 기대 값/화면 결과: 현재 actor, core action stage, 이동/경제 event, 후속 prompt가 `seq` 순서대로 이어져야 합니다.

### 수정할 때 주의할 점

- 규칙을 바꾸면 프론트 표시보다 엔진 테스트를 먼저 고치거나 추가하세요.
- 카드 이름 문자열보다 card id, trait, module metadata 같은 식별자를 우선 사용하세요.
- 즉시 효과인지, 대상 턴 시작에 해결되는 효과인지 구분하세요.

### 확인 방법

- `.venv/bin/python -m pytest GPT/test_rule_fixes.py -q`
- `.venv/bin/python -m pytest apps/server/tests/test_runtime_service.py -q`

### 초보자가 자주 헷갈리는 질문

- “이 값은 프론트가 계산하나요?”: 기본적으로 아니요. 규칙/권한/비공개성은 엔진과 백엔드가 결정해야 합니다.
- “화면이 맞으면 구현도 맞나요?”: 아니요. stream event, view_state, 테스트가 같은 흐름을 말해야 합니다.
- “어디부터 로그를 보나요?”: stream `seq`를 먼저 보고, 프롬프트 문제면 `request_id`, 런타임 문제면 `module_id`를 봅니다.
