보드 패널과 이벤트 오버레이가 말 이동을 표시합니다.

## 상세 인수인계

### 이 문서에서 이해해야 할 것

이 문서는 `주사위와 이동` 기능을 따라갑니다. 이동 값은 엔진이 결정하고, 프론트는 최종 위치와 move event를 화면에 반영합니다. 기능을 고칠 때는 한 계층만 보지 말고 엔진, 백엔드, 프론트 표시가 같은 흐름을 가리키는지 함께 확인해야 합니다.

### 관련 코드와 문서

- `GPT/runtime_modules/sequence_modules.py`
- `GPT/tile_effects.py`
- `GPT/runtime_modules/runner.py`
- `apps/server/src/services/runtime_service.py`

### 실제 처리 흐름과 기대 값

- 이 문서의 흐름: 주사위와 이동.
- 사용자 행동/트리거: 현재 턴의 이동 단계에서 주사위 카드나 기본 주사위가 확정될 때 발생합니다.
- 프론트 경로: 사용자 선택이 필요한 경우 `PromptOverlay`가 decision을 보내고, 자동 굴림이면 프론트 입력 없이 런타임이 진행합니다.
- 백엔드/엔진 경로: 엔진은 `sequence_modules.py`, 주사위/이동 관련 모듈, `tile_effects.py`를 거쳐 이동량과 경로를 확정합니다. 백엔드는 결과 이벤트와 최신 `view_state`를 stream으로 발행합니다.
- 기대 값/화면 결과: 기대 값은 `dice_roll`의 눈/합계, `player_move`의 출발/도착 칸, 보드 projection의 말 위치입니다. `BoardPanel`과 `useStepPathAnimation`은 서버가 준 경로를 표시해야 하며 프론트가 이동 규칙을 재계산하면 안 됩니다.

### 수정할 때 주의할 점

- 상위 문서와 하위 문서를 함께 읽고, 관련 테스트를 먼저 찾으세요.

### 확인 방법

- `python3 tools/plan_policy_gate.py`
- 관련 서버/프론트/엔진 테스트를 함께 실행하세요.

### 초보자가 자주 헷갈리는 질문

- “이 값은 프론트가 계산하나요?”: 기본적으로 아니요. 규칙/권한/비공개성은 엔진과 백엔드가 결정해야 합니다.
- “화면이 맞으면 구현도 맞나요?”: 아니요. stream event, view_state, 테스트가 같은 흐름을 말해야 합니다.
- “어디부터 로그를 보나요?”: stream `seq`를 먼저 보고, 프롬프트 문제면 `request_id`, 런타임 문제면 `module_id`를 봅니다.
